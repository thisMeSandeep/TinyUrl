### **1. Create Short Links**

* Input: long URL + optional custom shortcode.
* Output redirect URL:
  `https://your-site.com/<shortcode>`
* URL must be validated before saving.
* Custom codes must be **globally unique**.
* If a code already exists â†’ return **409 error**.

### **2. Redirect**

* Visiting `/<code>` performs an **HTTP 302 redirect** to the long URL.
* Each redirect must:

  * Increment **total click count**
  * Update **last clicked timestamp**

### **3. Delete a Link**

* Users can delete any link.
* After deleting:

  * Visiting `/<code>` must return **404**
  * No redirect should occur.

## **ðŸ“Œ API Specification**

Follow these EXACT endpoint names + behavior:

### **POST /api/links**

* Create a new short link
* Return **409** if shortcode already exists
* Validate URL

### **GET /api/links**

* Return a list of all short links

* Return stats for a specific code

* Delete the link
* After deletion, redirect should stop (404)
